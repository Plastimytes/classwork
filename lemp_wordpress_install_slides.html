01
System Preparation
Update the system
Sync Ubuntu's package manager with the latest software sources before installing anything.
sudo apt update && sudo apt upgrade -y

When prompted for a password, type the one you created at Ubuntu setup. No characters will appear — this is normal.

02
Web Server
Install & start Nginx
Nginx handles all incoming HTTP requests and serves your files to the browser.
sudo apt install nginx -y
sudo service nginx start
Verify
Open your Windows browser → http://localhost
You should see: Welcome to nginx!
03
Database
Install & secure MariaDB
MariaDB is your database server — a drop-in MySQL replacement that stores all WordPress data.
Install & start
sudo apt install mariadb-server -y
sudo service mariadb start
Secure
sudo mysql_secure_installation
# Press Enter for root pw
# Set a root password
# Answer Y to all prompts
04
PHP
Install PHP-FPM
Nginx doesn't process PHP directly — it passes requests to PHP-FPM, a dedicated process manager.
sudo apt install php-fpm php-mysql -y
php -v
# Note the version (e.g. 8.1)
sudo service php8.1-fpm start
# Replace 8.1 with your actual version
05
Nginx Configuration
Link Nginx to PHP
Edit the Nginx config to enable PHP processing, then test and restart the server.
sudo nano /etc/nginx/sites-available/default
Add / edit these blocks
index index.php index.html index.htm;

location ~ \.php$ {
  include snippets/fastcgi-php.conf;
  fastcgi_pass unix:/var/run/php/php8.1-fpm.sock;
}
sudo nginx -t && sudo service nginx restart
06
PHP Test
Verify PHP is working
Create a one-line test file to confirm Nginx and PHP-FPM are communicating correctly.
sudo nano /var/www/html/info.php
File contents
<?php phpinfo(); ?>
Verify
Open your browser → http://localhost/info.php
A large PHP configuration page should load.

502 Bad Gateway? Run sudo service php8.1-fpm start and check the socket version in your nginx config matches php -v.

07
WordPress · Database
Create the WordPress database
Provision an isolated database and a dedicated user account before downloading WordPress.
sudo mysql -u root -p
Run inside MariaDB prompt
CREATE DATABASE wordpress_db;
CREATE USER 'wp_user'@'localhost'
  IDENTIFIED BY 'your_secure_password';
GRANT ALL PRIVILEGES ON wordpress_db.*
  TO 'wp_user'@'localhost';
FLUSH PRIVILEGES;
EXIT;
08
WordPress · Files
Download & deploy WordPress
Fetch the latest WordPress package and move it into Nginx's public web root.
cd /tmp
wget https://wordpress.org/latest.tar.gz
tar -xvf latest.tar.gz
sudo mv wordpress /var/www/html/wordpress
Set permissions
sudo chown -R www-data:www-data /var/www/html/wordpress
sudo chmod -R 755 /var/www/html/wordpress
09
WordPress · Nginx
Point Nginx to WordPress
Update the server config so Nginx serves files from the WordPress directory with clean URL support.
sudo nano /etc/nginx/sites-available/default
Update root path
root /var/www/html/wordpress;
Update location / block
location / {
  try_files $uri $uri/ /index.php?$args;
}
sudo nginx -t && sudo service nginx restart
