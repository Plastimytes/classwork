Event delegation is a technique where you attach a single event listener to a parent element rather than individual children. When a child fires an event, it bubbles up to the parent, which handles it using event.target. This improves performance and works for dynamically added elements.
JavaScript is single-threaded. The event loop monitors the call stack and the task queue. When the stack is empty, it dequeues the next task. Microtasks (like Promise callbacks) always run before macrotasks (like setTimeout), giving JS its async-without-blocking character.
A closure is a function that retains access to variables from its outer lexical scope even after that scope has finished executing. It's how JavaScript enables data encapsulation and factory patterns. Every function in JS creates a closure over its surrounding context.
== is the loose equality operator — it performs type coercion before comparison ("1" == 1 is true). === is strict equality — it checks both value and type without coercion ("1" === 1 is false). Always prefer === unless you explicitly need coercion.
